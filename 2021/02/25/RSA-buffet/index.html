RSA-buffet

题目地址：https://adworld.xctf.org.cn/task/answer?type=crypto&number=5&grade=1&id=5544&page=5

这道题很有趣！

附件下载下来有5个密文，10个密钥，一个加密算法，一个明文生成算法。

先看下明文生成算法：generate-plaintexts.py，里面给出了关键算法secret-sharing的项目地址，https://github.com/blockstack/secret-sharing。

非常好，帮助我们理解第12行的split_secret()算法。但似乎项目的readme文档似乎没讲清楚是怎么split的，只说了“A library for sharding and sharing secrets (like Bitcoin private keys), using shamir’s secret sharing scheme.” 用shamir’s secret sharing scheme，类似比特币。继续搜索shamir’s secret sharing scheme。找到关于Shamir’s Secret Sharing https://blog.csdn.net/weixin_26722031/article/details/108136677

这篇文章讲的挺清楚的。

所以现在我们的情况就是拿到了这5个信封，放在ciphertext1～5里。是这样吗？nonono….还没这么简单。我们的明文的构成是怎样的看下generate-plaintexts代码：

所以也就是说，PLAINTEXTS[k]的组成是: message1+message 2(piece k)+ message 3（piece k） +message 4(piece k) +……+ message5(piece k)。应该是把5个PLAIINTEXTS带入encrypt.py里面加密后得到的5个ciphertext。

看看加密算法encrypt.py，是RSA和AES混合加密，算法里加密、解密的算法都已经给出。

这段是加密的算法，symmetric_key是AES的加密密钥，经过PKCS1_OAEP加密后，保存在密文的头部msg_header。PKCS1_OAEP是利用RSA进行的加密算法，和我们前面做的那个PKCS1_v1_5算法很类似。解密算法已经给出，但是需要RSA的密钥。

困惑的是，我们有5个密文，然而有10个公钥。先看看这10个公钥有几个能解出对应的私钥来。

先用RsaCtfTool工具把key-0.pem～key-9.pem对应的prikey生成出来。类似这样的命令：

$ python3 RsaCtfTool.py --private --publickey key-3.pem --output prikey-3.pem

一波下来，key0、1、2、3、6对应的密钥prikey-0、1、2、3、6可以解出来。

先写个exp把这几个私钥代入解密函数里面看看能否解的出来。

发现结果可喜！每个ciphertext（除了2）都有一个prikey可以解开。而且格式符合split_secret结果的格式。Congratulations, you decrypted a ciphertext! One down, two to go :) 每个plaintext都有的这句话应该就是message1的内容。

虽然ciphertext2没解开，但是shamir算法要求只要找到的pieces数量能达到阈值就可以正确还原message 了。阈值在哪呢？generate-plaintexts代码里的shares = SS.split_secret(msg, i, 5) ，i 即是阈值。这是通过github该项目里面的secret-sharing这个文件里面找到的。

所以 i 是阈值，即message2需要2个pieces，message3 需要3个pieces …….所以flag5要5个piece才能完全恢复，我们目前只有4个。悲……….

还是再写个exp把前面几个message恢复再说。在https://github.com/blockstack/secret-sharing readme文档里面已经讲的很清楚怎么恢复文档了。这里我们要选PlaintextToHexSecretSharer这个方法，能直接还原出文字。这里我卡了比较久，因为我用SecretSharer这个方法了，解出来是16进制，再转换成字符串就都是乱码。没注意到有这个可以直接转化成文字的方法。。。。。。下次阅读文档真的要细心一些。

特别注意，每一个share都要从前面每一个PLAINTEXTS 里面找出1- 3- 4- 5-开头的串。因为我们的PLAINTEXTS[k]的组成是: message1+message 2(piece k)+ message 3（piece k） +message 4(piece k) +……+ message5(piece k)。
