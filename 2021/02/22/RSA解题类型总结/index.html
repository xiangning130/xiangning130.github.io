RSA解题类型总结

Estimated Reading Time 7 Minutes

这篇文章着重总结一下RSA算法在CTF比赛中常出现的题目类型。

一、基本题型：

这类题目最简单，已知n、e、c求明文m，或已知公钥、密文求明文。

CTF中一般把flag作为明文，在RSA里面就是用m表示。c是密文，n和e共同组成公钥。n和d共同组成私钥。n和e一般是已知的。

加密过程：c=m^e mod n

解密过程：m=c^d mod n

求解私钥d

Crypto 包里的 inverse() 函数, 两个参数不互素的时候返回的是除以最大公因数之后的逆元。

使用 gmpy2 包里的 invert() 函数，两个参数不满足互素时会报错，只有满足互素时正常求逆元。

这里涉及到p和q的求解，是破解RSA的关键。因为N=p*q，p和q为两个大素数。RSA的安全性就是在于对与一个大整数而言，求其质因子是非常困难的。

我们可以通过一些工具来帮助，如在线因子分解网站、yafu、sage等。

二、特殊算法攻击手法

当n很大尝试了多种手段也无法分解时，这时就要利用题目给出的其他条件用特殊算法进行攻击了。

1、共模攻击

条件

如果条件给出了如果有两组待解密文件和key文件，且两组的模N都相同，可以用共模攻击，在不分解n的情况下解出明文。

解题

用CTF-RSA-Tools工具里实现了共模攻击的算法：将两组的N、e、c写入share_N.txt，运行：

$ python solve.py --i share_N.txt

2、公因数攻击

条件

如果条件给出有两组n、e、c，且n不相同，n很大很难分解，可以试试公因数攻击。也就是多对公钥n是公用了一个素数因子。

解题

攻击脚本。在RSA/exp/gcd_exp.py
